Collatz by histogram


An integer can be represented in binary notation. The binary notation is a finite string of 1's and 0's.

An alternate description of the Collatz sequence is "next element is one plus the previous element multiplied then remove the trailing zeroes". Multiplying by 3 is achievable by shifting the previous element up one and adding to the unshifted prior value, and the "one plus" part achievable by setting the lsb of the shifted number to 1. The Collatz conjecture is that regardless of initial value a Collatz sequence will go to 1 in a finite number of steps.

A binary formatted number can be recorded as a set of pairs of integers listing the number of contiguous ones and the number of zeroes separating that group of ones from the next more significant group. An example:
159310= 110001110012= 1:2,3:3,2:0 where 0 marks that pairing as the most significant one.
159910= 110001111112= 6:3,2:0.
Introducing some Notation:
n:z is a 'n' contiguous ones paired with 'z' contiguous zeroes.
N(x)= number of pairs whose 'number of ones' equals x.
Z(x) = number of pairs whose 'number of zeroes' equals x.

The Collatz conjecture as described herein says that the Collatz sequence yields a single pair whose value is 1:0 (=110) in a finite number of steps regardless of the value initial value. Alternately worded that after some finite number of steps N(1)=1, N(>1)=0.

The conjecture is proven by noting all the finite possible cases of transformations of the set of N(x),Z(x) values per each step, and showing how the sum of the N(x) values over all x increases and decreases as the sequence is followed in such a fashion that it becomes 1 in a finite number of steps.

For our example number of 159310 the array of N(x) values in ascending order (starting at 1) is: 1,1,1. Note that the N() histogram is not unique, different numbers can have the same histogram of N values.

There are two special pairs: the most significant one, whose number of zeros is 0 or otherwise irrelevant; and the least significant one, which suffers a subset of the cases that intermediate pairs do. For conciseness I will call the least significant pair the trailing pair, the most significant pair the leading pair.

For values of N are interesting:
  1. N(1), the number of isolated 1 bits.
  2. N(2), the number of isolated pairs of 1 bits.
  3. The sum of all remaining N's: Nother.
  4. The sum of all of the above: Ntotal.

We will explore how the above numbers change at each step of the sequence.
